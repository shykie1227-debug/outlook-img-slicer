# 一键打包脚本使用说明

## 📦 快速开始

**双击 `一键打包.bat` 即可开始打包，全程无需任何手动操作！**

---

## ✨ 功能特性

### 全自动流程

| 步骤 | 功能 | 说明 |
|------|------|------|
| 1 | 备份关键配置 | 自动备份 package.json、electron-builder.yml 等配置文件 |
| 2 | 清理冗余文件 | 自动识别并删除旧版本打包脚本、缓存文件 |
| 3 | 环境检测 | 自动检测 Node.js、Python 是否安装 |
| 4 | 国内镜像 | 自动配置 npm / Electron 国内镜像，加速下载 |
| 5 | 依赖安装 | 自动安装 Node.js 和 Python 依赖 |
| 6 | 构建渲染进程 | 自动构建 React 前端 |
| 7 | 构建主进程 | 自动构建 Electron 主进程 |
| 8 | 打包 Sidecar | 自动打包 Python Sidecar |
| 9 | 打包 EXE | 自动打包 Windows 绿色免安装版 |
| 10 | 验证结果 | 自动验证构建产物并生成报告 |

### 无需配置

- ❌ 不需要输入任何参数
- ❌ 不需要手动设置环境变量
- ❌ 不需要手动安装依赖
- ✅ 双击即运行，运行即成功

## 🚀 使用方法

### 方法一：双击启动（推荐）

1. 打开项目文件夹
2. **双击 `一键打包.bat`**
3. 等待打包完成（约 5-10 分钟）
4. 打包完成后按任意键退出

### 方法二：PowerShell 运行

```powershell
# 进入项目目录
cd "项目路径"

# 运行脚本
.\一键打包.ps1
```

## 🔧 自动清理内容

脚本会自动清理以下内容：

### 旧版本文件
- `build_v5*.py` - V5 版本 Python 构建脚本
- `build_v5*.ps1` - V5 版本 PowerShell 脚本
- `*.spec` - PyInstaller 配置文件
- 其他过时的打包相关文件

### 缓存文件
- `__pycache__/` - Python 字节码缓存
- `.pytest_cache/` - pytest 缓存
- `app/dist/` - 旧的渲染进程构建
- `electron/dist-electron/` - 旧的主进程构建
- `sidecar/dist/` - 旧的 Sidecar 构建
- `sidecar/build/` - 旧的 Sidecar 中间文件
- `release-artifacts/electron/` - 旧的打包产物

### 不会清理的内容
- `node_modules/` - Node.js 依赖（保留加快速度）
- `legacy/` - 遗留代码目录
- `docs/` - 文档目录
- `tests/` - 测试文件
- 用户源代码文件

## ❓ 常见问题

### Q: 双击后一闪而过怎么办？

A: 请右键选择「以管理员身份运行」，或者在 PowerShell 中运行脚本查看错误信息。

### Q: 提示找不到 Node.js？

A: 请先安装 Node.js 18+，安装时务必勾选 "Add to PATH"，安装完成后重启电脑。

### Q: 提示找不到 Python？

A: 请先安装 Python 3.10+，安装时务必勾选 "Add Python to PATH"。

### Q: npm install 很慢/超时？

A: 脚本已自动配置国内镜像，如仍有问题请检查网络连接。

### Q: 打包失败怎么办？

A: 
1. 查看同目录下的 `打包日志_*.log` 文件
2. 查看最后几行的错误信息
3. 确保已安装 Node.js 和 Python
4. 尝试删除 `node_modules` 后重新运行

### Q: 可以重复运行吗？

A: 可以！每次运行都会清理旧的构建产物，重新开始打包。

### Q: 如何只重新打包，不重新安装依赖？

A: 依赖只在首次运行时安装，后续运行会自动跳过安装步骤。
