# Design System: Outlook 长图助手
**Project ID:** Local PySide6 Desktop Application (No Stitch Project)

## 1. Visual Theme & Atmosphere

界面采用安静、可信、轻量的 Windows 桌面工具气质。视觉重点不是装饰，而是让非技术用户一眼看懂“放入文件 → 自动切图 → 添加链接 → 创建 Outlook 邮件”的顺序。整体保持浅色、宽松、低干扰，避免网页仪表盘式的卡片堆叠和过度动效。

信息层级遵循三个层次：顶部说明用途，中部完成文件与参数操作，底部提供明确的主要动作。未满足执行条件时，控件仍需保持可辨识，不能因灰度过低而像“消失”。

## 2. Color Palette & Roles

- **Outlook 行动蓝 (#0078D4)：** 主要按钮、焦点边框、滑块与可交互链接，建立与经典 Outlook 一致的信任感。
- **深海军文本 (#111827)：** 标题与关键数据，承担最高阅读层级。
- **冷灰正文 (#6B7280)：** 辅助说明、参数单位、次级状态。
- **雾白背景 (#F8F9FA)：** 应用主背景，降低长时间操作的视觉负担。
- **纯白工作面 (#FFFFFF)：** 拖放区、输入框、预览容器与次级按钮。
- **柔和描边灰 (#E1E4E8)：** 容器边界和控件分隔，不抢夺内容注意力。
- **经典 Outlook 深蓝 (#005A9E)：** 主要按钮按下状态。
- **可靠成功绿 (#0E7B3C)：** 完成、链接已添加、文件已保存等确认状态。
- **克制警告橙 (#D83B01)：** 邮件过大、兼容性提醒和需要用户检查的状态。
- **清晰错误红 (#C4314B)：** 文件损坏、处理失败和无法启动 Outlook。

## 3. Typography Rules

Windows 使用 Microsoft YaHei UI 或 Microsoft YaHei，其他平台测试时回退到系统无衬线字体。主标题使用 20px 粗体；区块标题和主按钮使用 13–14px 中粗体；正文和控件使用 12–13px；版本与辅助说明使用 10–11px。

文案使用短句和动词开头。状态信息只保留一个语义图标，避免“✅ ✅”等重复符号。专业词后附用户能理解的解释，例如“避开文字切图（推荐）”而不是只写“智能切图”。

## 4. Component Stylings

* **Buttons:** 主要按钮使用 Outlook 行动蓝、白色文字和 8px 柔和圆角；次级按钮为白底描边；文字按钮仅用于重置和低风险辅助操作。禁用状态保持足够对比度，并通过明确条件提示告诉用户如何启用。
* **Cards/Containers:** 拖放区使用 12–14px 柔和圆角与浅灰虚线边框。预览区使用白底、1px 描边，不使用网页式大阴影；原生 Qt 不使用不受支持的 `box-shadow`。
* **Inputs/Forms:** 输入框使用白底、1px 柔和灰边框，聚焦时切换为 Outlook 行动蓝。邮件宽度显示用途和推荐值，避免只出现孤立数字。
* **Status:** 状态栏同时说明当前阶段和下一步。成功、警告、错误分别使用绿色、橙色、红色，但不只依赖颜色表达含义。
* **Hotspot Editor:** 框选区域、链接输入和已添加列表保持同一操作语义；添加后必须显示链接数量和所在切片，删除或修改后立即刷新。
* **Cut Editor:** 使用可滚动的整图预览和高对比横向切线。拖动时显示像素位置；非法位置直接被限制，不等用户点击确定后才报错。保留“恢复自动切线”和“取消”作为安全出口。

## 5. Layout Principles

保留现有纵向桌面结构，不重构为网页式多栏界面。顶部 24px 外边距，中间控件间距以 8–12px 为主，区块之间使用 16–24px 呼吸空间。

文件未载入时，拖放区是视觉中心；文件处理后，预览、“调整切图位置”和“添加可点击按钮”成为中间焦点；底部始终由“在经典 Outlook 中创建邮件”承担唯一主操作。复制 HTML 和保存切图作为明确的备用路径，不能与主操作争夺视觉优先级。
