# Outlook 长图无损插入工具 — UI 布局规范分析报告

> 文档版本：1.0 | 分析日期：2026-05-11 | 分析范围：main.py (720×780 主窗口)

---

## 1. 当前 UI 问题诊断

### 1.1 视觉层级问题

| 问题 | 具体表现 | 严重程度 |
|------|---------|---------|
| **标题字号被降级** | Header 使用 18px Bold，视觉上与副标题（12px gray）拉不开足够差距；副标题 12px 过小，与正文 13px 差距微弱 | 🔴 高 |
| **副标题灰度不够** | `color: #6B7280` 在浅灰背景上辨识度不足，尤其在表单密集区域容易被"淹没" | 🟡 中 |
| **调试标签与标题争视线** | BUILD 标签是绿色小标签，位置在 Header 上方 16px 外，独立成行。高饱和度绿色与 Header 标题同等视重，打断视觉流 | 🔴 高 |
| **按钮文字层级混乱** | "创建 Outlook 邮件"用 14px Bold，"保存切图"用 13px Normal；"选择文件"和"重置"也是 13px——主按钮字号差只有 1px，层次感极弱 | 🔴 高 |

### 1.2 间距 / 呼吸感问题

| 问题 | 具体表现 | 严重程度 |
|------|---------|---------|
| **整体间距偏紧** | `layout.spacing=14`，容器 `margin=20,16,20,16`，在 720px 宽度下显得拥挤；元素间缺乏明确的分组感 | 🟡 中 |
| **行间距未分组** | Row1（选择文件/重置/宽度）与下方收件人输入框之间无视觉分隔，两组不同语义的元素紧贴在一起 | 🟡 中 |
| **底部按钮行间距过大** | `bottom_row.setSpacing(16)` 配合两侧按钮，视觉上形成"两个独立主按钮"错觉，而非一个主+一个次的层次 | 🟡 中 |
| **调试标签区 margin 过大** | 调试标签行单独占一行，`layout.setSpacing=14`，标签上方没有其他元素，却用了与其他区域相同的间距体系，显得孤立 | 🟢 低 |

### 1.3 按钮层次问题

当前按钮现状：

```
[创建 Outlook 邮件]  ← 主按钮：48px高，14px Bold，PRIMARY色，固定宽度
[保存切图]           ← 次按钮：默认高度，13px Normal CARD色，有边框
[选择文件] [重置]    ← 辅助按钮：13px Normal，CARD色+边框
```

**问题：**
- "创建 Outlook 邮件"与"保存切图"视觉重量相近（均有圆角+阴影感），主次区分不够明显
- "保存切图"用 `Theme.CARD` 白底灰字，视觉权重几乎等同于"重置"按钮，层次模糊
- Primary 按钮使用 `#0078D4`（微软蓝），但在 12px 副标题+13px 表单+14px 主按钮的字体体系中，Primary 色没有承载足够的语义等级

### 1.4 色彩 / 主题一致性

| 问题 | 具体表现 | 严重程度 |
|------|---------|---------|
| **Theme 类不完整** | 缺少：辅助色、禁用状态色、hover 分层、透明色规范 | 🟡 中 |
| **Primary 蓝与成功绿并置** | Primary=`#0078D4`，Success=`#10B981`，调试标签用 `#10B981` 绿色。在同一个界面中两种高饱和色并用，显得廉价 | 🟡 中 |
| **输入框状态缺失** | 只有 default/border-primary（focus）两种状态，没有 hover 状态、没有 error 状态 | 🟢 低 |
| **BORDER 色过重** | `BORDER=#E5E7EB` 在白底卡片上偏重，形成明显的分割感；现代设计倾向使用更轻的分隔线 | 🟡 中 |

## 2. 优化后的布局规范

### 2.1 间距系统

```
主容器 margin:      24px (左/右), 20px (上), 16px (下)
主布局 spacing:     12px（元素间）/ 20px（语义组之间）
按钮组间距:         12px（primary + secondary）/ 8px（辅助按钮之间）
语义分组间距:       每 20px 一个视觉分组（类似 Figma 的 section spacing）
```

**分组原则（从顶到底）：**
```
[调试标签行]        gap: 0px（不占间距预算，因为它独立成行）
     ↓ 12px
[标题 + 副标题]     gap: 6px（组内）
     ↓ 20px（语义分组）
[拖放区]            gap: 0px（自身完整）
     ↓ 20px（语义分组）
[Row1: 按钮+宽度]    gap: 12px（内部元素）/ 与下方间距 16px
     ↓ 16px
[收件人/标题]       gap: 12px（两行输入框）
     ↓ 20px
[预览区]
     ↓ 12px
[进度条]
     ↓ 8px
[状态文字]
     ↓ 20px
[底部按钮行]
     ↓ 12px
[版本]
```

### 2.2 字体层级

|------|------|------|------|------|------|---------|
| 调试标签 | 10px | 删除（上线前） | — | — | — | — |
| 主标题 | 18px Bold | 保持或改为 20px Bold | 20px | Bold (700) | `#111827` | - |
| 副标题 | 12px gray | 保持但提高灰度 | 13px | Regular (400) | `#6B7280` → `#9CA3AF` | 与标题间距 6px |
| 表单标签文字 | —（placeholder） | 13px | 13px | Regular | `#9CA3AF` | — |
| 表单输入 | 13px | 保持 | 13px | Regular | `#111827` | — |
| 按钮 Primary | 14px Bold | 保持 | 14px | Bold (700) | white | — |
| 按钮 Secondary | 13px | 保持 | 13px | Medium (500) | `#374151` | — |

### 2.4 色彩规范（扩展 Theme 类）

```python
class Theme:
    # 主色
    PRIMARY = "#0078D4"           # 主操作按钮、进度条激活状态
    PRIMARY_HOVER = "#2563EB"     # 主操作 hover（更深蓝）
    PRIMARY_ACTIVE = "#1D4ED8"    # 主操作 active（最深蓝）
    PRIMARY_DISABLED = "#BFDBFE"  # 主操作禁用（浅蓝）

    # 辅助色
    SECONDARY = "#F3F4F6"         # 次按钮背景
    SECONDARY_HOVER = "#E5E7EB"   # 次按钮 hover
    SECONDARY_TEXT = "#374151"     # 次按钮文字

    # 功能色
    SUCCESS = "#10B981"           # 成功状态（仅状态文字，不做标签）
    WARNING = "#F59E0B"           # 警告状态
    ERROR = "#EF4444"             # 错误状态

    # 中性色（新版）
    BG = "#F8FAFC"                # 页面背景
    CARD = "#FFFFFF"              # 卡片/输入框背景
    BORDER = "#E5E7EB"            # 默认边框
    BORDER_FOCUS = "#0078D4"      # 聚焦边框
    BORDER_HOVER = "#D1D5DB"      # hover 边框

    TEXT_PRIMARY = "#111827"      # 主要文字
    TEXT_SECONDARY = "#6B7280"    # 次要文字（当前用这个，建议降）
    TEXT_PLACEHOLDER = "#9CA3AF"  # 占位符/弱提示文字
    TEXT_DISABLED = "#D1D5DB"     # 禁用状态文字

    # 拖放区
    DROPZONE_IDLE_BORDER = "#D1D5DB"   # 闲置虚线边框
    DROPZONE_HOVER_BG = "#EFF6FF"     # hover 背景（极浅蓝）
    DROPZONE_HOVER_BORDER = "#0078D4"  # hover 边框
```

## 4. 具体改进建议（按优先级）

### 🔴 P0 — 必须修复（影响核心体验）

#### P0-1：删除调试标签行（或将其折叠为日志模式）

**问题**：BUILD 标签在视觉上打断了标题区，让用户困惑。

**方案 A（推荐）**：完全删除，替换为 About 对话框中的版本信息。

**方案 B**：保留但改为灰色文字，置于窗口标题栏（或 Footer 区域，不占用主布局）。

#### P0-2：区分 Primary 与 Secondary 按钮视觉权重

**当前**：两个底部按钮视觉重量相近。

**修改**：
- "创建 Outlook 邮件"：保持 Primary 样式（48px, 14px Bold, 主蓝底白字）
- "保存切图"：降为 Secondary 样式（40px, 13px Medium, 白底灰字灰边框），或在 Primary 按钮下方单独一行

#### P0-3：统一输入框高度

**当前**：`input_to` 和 `input_subject` 用 36px，但 SpinBox 内的输入框视觉上更高。

**修改**：统一为 40px，与 Secondary 按钮同高，视觉更整齐。

---

### 🟡 P1 — 重要改进（提升整体品质）

#### P1-1：细化副标题灰度

**当前**：副标题 `color: #6B7280`，在灰色背景上辨识度低。

**修改**：副标题降为 `#9CA3AF`，主标题升为 `#111827`，对比度从 3.2:1 提升到 5.8:1。

#### P1-2：重构字体层级，主标题加粗加大

**当前**：18px Bold 与副标题 12px 差距不够。

**修改**：主标题升至 20px Bold，副标题 13px Regular，`line-height` 间距 6px。

#### P1-3：统一按钮圆角体系

**当前**：主按钮圆角 12px，次按钮 10px，辅助按钮 10px，不统一。

**修改**：建立三级圆角规范 `12px / 8px / 6px`，用于不同层级按钮。

#### P1-4：扩展 Theme 类（增加禁用色、hover 色、active 色）

当前 Theme 类只有 8 个颜色，建议扩展到 15+，覆盖所有组件状态。

---

### 🟢 P2 — 优化建议（可选）

#### P2-1：调整容器 margin，窗口收紧 20px

当前 `margin=20,16,20,16`，改为 `margin=24,20,24,20`（左右增加 4px，呼吸感更好）；同时窗口高度从 780px 降至 760px。

#### P2-2：预览区增加空状态提示

当前预览区为空时直接 `hide()`，用户不知道这里将来会显示什么。建议空状态显示一行灰色小字："切片预览区"。

#### P2-3：Row1 按钮组增加图标

"选择文件" → "📂 选择文件"，"重置" → "🔄 重置"，提升可发现性。
