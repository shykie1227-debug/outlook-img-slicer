# Outlook 经典版长图可靠性与操作体验设计

## 目标

让非技术用户打开工具即可完成图片/PDF/PPT/PSD 切图、添加按钮链接、创建经典 Outlook 邮件，并保证长图内容完整、热区布局稳定、复制 HTML 不因中文或长度而截断。

## 设计选择

采用“最小风险修复 + 关键边界收敛”，不进行全项目分层重写：

1. 保留普通图片的 V3 连续 `<img>` 路径，它已有历史无缝基线。
2. 热区按 Y 行分段，每个行段只使用当前活跃按钮的 X 边界，避免全局网格产生无意义碎片。
3. 每个热区行独立输出 inline segment，不共享 Outlook 表格列。
4. 经典 Outlook 主路径继续使用 CID 附件并只调用 `mail.Display(False)`。
5. 复制 HTML 继续提供 CF_HTML，但所有偏移严格按 UTF-8 字节计算。

## 核心数据流

原始文件先被渲染并切成不高于 1200px 的邮件切片。用户添加热区后，每张受影响切片按热区上下边界拆成视觉行；无按钮行保持一张完整图片，有按钮行只沿本行按钮边界拆分。每个物理片段携带明确的 `sort_key`、链接和替代文字。

生成 Outlook 邮件时，片段转为 CID 附件并按 `sort_key` 绑定。复制 HTML 时，同一结构使用 Base64 图片并封装为 Windows `HTML Format`。两条路径共享同一分组和布局规则，避免“发送正常、复制错位”。

## 操作体验

- 默认邮件宽度调整为 Outlook 验证基线 650px，并明确标记“推荐”。
- “智能切图”改为“避开文字切图（推荐）”。
- 主按钮改为“在经典 Outlook 中创建邮件”。
- 自动切图后提供“调整切图位置”：在整图预览上拖动横向切线，并可恢复自动切线。
- 主界面加入“1 放入文件 → 2 调整切线/添加链接 → 3 创建邮件”的常驻轻量引导。
- 文件处理后显示切片数量、可点击按钮数量和预计邮件大小。
- 状态栏统一添加一次语义图标，调用方不再重复嵌入图标。
- 删除 Qt 不支持的阴影样式，避免运行时样式警告。

## 错误与恢复

- 热区真实重叠继续阻止并给出可执行提示；上下错开的按钮允许共存。
- 手动切线必须保持严格递增，每片至少 80px 且不超过 1200px。
- 已添加热区后再次调整切线，必须先明确提示热区会被清空，防止链接落到错误图片区域。
- 手动调整不会裁掉、重复或重排任何像素；取消对话框不改变当前切片。
- CF_HTML 生成后校验四个偏移均指向正确字节位置。
- 热区切片数量按最小分段策略生成，避免长图附件数指数膨胀。
- Outlook 自动化失败时保留现有经典版/新版区分提示。
- 所有临时片段继续本地保存和清理，不引入网络、云 API、上传或自动发送。

## 测试策略

- 单元测试验证 CF_HTML 中文、ASCII、完整 fragment 边界。
- 单元测试验证两个上下错开的按钮从 25 个片段降至 9 个，并保持两个链接。
- 单元测试验证手动切线合法性、像素完整性、安全高度和恢复自动切线。
- PySide6 离屏测试验证调整按钮启用条件、功能引导和热区清空防线。
- HTML 结构测试锁定普通路径单表单元格和热区路径独立行。
- PySide6 离屏测试验证默认宽度、按钮文案、状态图标和控件可用状态。
- 完整回归执行全部 pytest、Python 编译检查、离线代码审计、HTML 浏览器视觉检查和原生窗口截图检查。

## 非目标

- 不自动发送邮件。
- 不为新版 Outlook 增加登录或云端集成。
- 不重构整个 GUI 为多页面或网页界面。
- 不承诺在 macOS 上验证 Outlook COM；最终 EXE 仍由用户在 Windows 本地构建。
