# V4.9.1 热区功能临时屏蔽日志 (2026-06-18)

## 触发

用户反馈：Outlook 再检查一次切图后，拼接中间仍有缝隙；要求先屏蔽隐藏“添加可点击按钮”功能。

## 根因假设

当前拼接缝排查的最大变量是 hotspot / 可点击按钮路径：

1. 用户添加可点击按钮后，原始切片会进入 `hotspot_slicer.py` 物理切割。
2. HTML 组装会把多个派生切片重新排序、链接包裹、拼接。
3. Outlook Word 引擎对多段图片、链接 `<a>` 包裹和表格/块级堆叠的渲染存在 1px 缝隙风险。

因此 V4.9.1 先把热点功能从用户路径中彻底隔离，保留普通切图/复制 HTML/创建 Outlook 邮件，用于判断普通切图链路是否仍有缝隙。

## 修改内容

### `main.py`

- 版本号：`4.9.0` → `4.9.1`
- 新增功能开关：

```python
HOTSPOT_FEATURE_ENABLED = False
```

- 工具栏中不再添加/显示“🎯 添加可点击按钮”。
- 缩略图不再显示“✏️ 编辑热区”。
- 缩略图不再绑定打开热区编辑器的点击事件。
- `_open_hotspot_editor()` 增加防线：开关关闭时只提示，不打开编辑器。
- `_build_slices_with_hotspots()` 增加防线：开关关闭时完全忽略历史 `hotspot_map`，直接返回普通 `SliceItem`，`href=None`。
- 保留普通切图、保存切图、复制 HTML、创建 Outlook 邮件、压缩发送功能。

### `tests/test_v491_hotspot_disabled.py`

新增 3 个回归测试：

1. `HOTSPOT_FEATURE_ENABLED is False`
2. 按钮不可见，且历史 hotspot 数据不会进入发送/复制路径
3. 缩略图在功能关闭时没有热区点击 handler

## 风险

- 🟡 临时禁用了“添加可点击按钮”功能，用户无法在 V4.9.1 使用热区链接。
- 🟢 普通切图/保存/复制/创建 Outlook 邮件不受影响。
- 🟢 通过开关可恢复，不需要删除 hotspot 相关代码。

## 回滚/恢复方式

如需恢复可点击按钮功能：

```python
HOTSPOT_FEATURE_ENABLED = True
```

恢复前必须重新验证 Outlook 中 hotspot 链路是否仍产生拼接缝。

## 下一步

1. 构建 V4.9.1 EXE。
2. Windows 本地磁盘运行，不从 `\\Mac\Home` 直接运行。
3. 用普通切图链路创建 Outlook 邮件。
4. 若普通切图仍有缝，则继续排查 HTML 纵向堆叠/图片物理高度/Word 缩放。
5. 若普通切图无缝，则确认 hotspot 链路是主因，再单独重构可点击按钮。
