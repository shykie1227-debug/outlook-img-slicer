# 架构：V4.6.7 显式 sort_key 排序

> V4.6.7 引入  ·  解决"顺序错误 / 错位"类问题  ·  适用范围：所有切图路径

---

## 背景

### V4.6.6 顺序问题

V4.6.6 物理切割后，HTML 输出顺序依赖**隐式状态**：

1. 智能切图产物 `image_slicer.detect_and_slice` 的返回顺序
2. `hotspot_slicer.slice_paths_by_hotspots` 的 append 顺序
3. `html_assembler` 的 for 循环顺序
4. 未来可能引入的 OCR / 文本切片 → 顺序来源越多越乱

用户报告："切图后顺序不正确"。

## V4.6.7 解决方案：显式 sort_key 字段

### 数值层级

```
原切片:    sort_key = N.000   (N = 1, 2, 3, ...)
Hotspot 派生竖条: sort_key = N.M   (M = N*0.001, N=1, 2, ..., 999)
```

**例**：1 张原图标 2 个 hotspot →
```
1.000  ← 原始切片 1
1.001  ← Hotspot 派生自切片 1，第 1 个竖条
1.002  ← Hotspot 派生自切片 1，第 2 个竖条
2.000  ← 原始切片 2
...
```

### 唯一排序点

```python
# html_assembler.assemble_html
sorted_slices = sorted(slices, key=lambda s: s.sort_key)
# 之后所有逻辑只用 sorted_slices，不再依赖输入顺序
```

## 架构红线（禁止的依赖）

| 维度 | 是否允许 |
|------|----------|
| 依赖 append 顺序 | ✗ 禁止 |
| 依赖文件名排序 | ✗ 禁止 |
| 依赖目录遍历（os.listdir / glob）顺序 | ✗ 禁止 |
| 依赖 `image_paths.index()` 兜底 | ✗ 禁止（O(N) 性能 + 重复路径 bug） |
| 依赖 sort_key 字段 | ✓ 唯一允许 |

## 修复的 5 个 V4.6.7 隐藏 bug

V4.6.8 专家审查发现的 5 个 bug 都在 sort_key 透传路径上：

### 隐藏根因
`Hotspot.normalized()` 不传 source_index
```python
# 修复前
def normalized(self):
    x1, x2 = sorted((self.x1, self.x2))
    y1, y2 = sorted((self.y1, self.y2))
    return Hotspot(x1=x1, y1=y1, x2=x2, y2=y2, url=self.url, text=self.text)
    # ❌ 漏了 source_index=self.source_index

# 修复后
def normalized(self):
    ...
    return Hotspot(
        x1=x1, y1=y1, x2=x2, y2=y2,
        url=self.url, text=self.text,
        source_index=self.source_index,  # ✓ 透传
    )
```

### 1. `HotspotMap.update()` 不透传 source_index
```python
# 修复前
h = Hotspot(..., url=url, text=text)  # ❌ 缺 source_index

# 修复后
source_index = hotspot.source_index or lst[index].source_index  # 保留原值
h = Hotspot(..., source_index=source_index)  # ✓
```

### 2. `HotspotMap.from_json()` 不透传 source_index
```python
# 修复前
ok, _ = m.add(k, Hotspot.from_dict(h))

# 修复后
hs = Hotspot.from_dict(h)
ok, _ = m.add(k, hs, source_index=hs.source_index)  # ✓
```

### 3. `slice_paths_by_hotspots` 兜底用 O(N) `.index()`
```python
# 修复前
for path in image_paths:
    source_index = float(image_paths.index(path) + 1)  # O(N) + 重复路径 bug

# 修复后
for idx, path in enumerate(image_paths):  # O(1) enumerate
    source_index = float(idx + 1)
```

### 4. `reset_app()` 不重置 `slice_source_index`
```python
# 修复前
def reset_app(self):
    self.slice_paths = []
    self.file_path = None
    # ❌ 缺：self.slice_source_index = {}

# 修复后
def reset_app(self):
    self.slice_paths = []
    self.slice_source_index = {}  # ✓ 同步清空
```

## 数据流（V4.6.7 排序架构）

```
1. 智能切图 image_slicer.detect_and_slice
   返回 List[path]  ← 约定 = 用户视觉顺序（按 Y 自上而下）

2. _on_processed (main.py)
   ↓
   self.slice_source_index = {fname: float(i+1) for i, p in enumerate(paths)}
   ↓  # 1.0, 2.0, 3.0, ...

3. 用户标注 hotspot
   ↓
   HotspotMap.add(slice_filename, hotspot, source_index=N.0)
   ↓
   Hotspot.source_index = N.0  (持久化)

4. 发送/复制 HTML (main.py._build_slices_with_hotspots)
   ↓
   slices_with_key, link_map = slice_paths_by_hotspots(
       paths, hotspots_by_slice, source_index_map=self.slice_source_index
   )
   ↓
   物理切割：
     - 原切片 sort_key = N.000
     - 派生竖条 sort_key = N.M (M = N*0.001)
   ↓
   List[(path, sort_key)]

5. main.py 转 List[SliceItem]
   ↓
   items.append(SliceItem(path=p, sort_key=sk, original_width=orig_w))

6. html_assembler.assemble_html
   ↓
   sorted_slices = sorted(items, key=lambda s: s.sort_key)  # ✓ 唯一排序点
   ↓
   同一 source_index 的多段横向拼接 1 个 <tr>
   不同 source_index 之间纵向堆叠
   ↓
   HTML
```

## 边界 case 处理

### 智能切图切了 N 段 Y（按行）
```
source_index 1.000  ← 顶部 1 段
source_index 2.000  ← 中部 1 段
source_index 3.000  ← 底部 1 段
```

→ 邮件里 3 个 `<tr>`（每行 1 张原图），行内按 hotspot 派生竖条

### 用户标 1 个 hotspot
```
1.000  ← 原切片 1
1.001  ← 左竖条（普通）
1.002  ← 中竖条（链接）
1.003  ← 右竖条（普通）
```

→ 1 个 `<tr>` 含 3 个 `<td>`

### 多张原图 + 多 hotspot
```
1.000, 1.001, 1.002  ← 原图 1（3 段）
2.000, 2.001         ← 原图 2（2 段）
3.000, 3.001, 3.002, 3.003  ← 原图 3（4 段）
```

→ 3 个 `<tr>`，每行内多 `<td>` 横向拼接
