# 架构：V1 物理切割

> V4.6.6 引入  ·  解决 Outlook Desktop 不可点  ·  适用范围：图片格式

---

## 背景

### V4.6.5 之前的方案（`<map>+<area>`）

```html
<map name="hotspots_001">
  <area shape="rect" coords="x1,y1,x2,y2" href="https://example.com" alt="链接" />
</map>
<img src="cid:slice_001" usemap="#hotspots_001" />
```

**问题**：Outlook Desktop 用 Word 引擎渲染邮件，**不解析 `<map>` 和 `<area>`** → 桌面端用户点不动图上的链接。

### 其他失败尝试
- **V4.6.5 试错**：图外加可点击文字链接（"双保险"）→ 用户反馈"看着像占位符，没有按钮感"
- **V4.6.6 试错**：图外加按钮组（V4.6.6 按钮方案，已被本次 V1 覆盖）→ 用户报告"邮件里有额外元素"
- **透明 `<a>` 叠层**：Outlook Desktop 忽略 `position: absolute` → 不可点

## V1 物理切割方案

### 核心思想

**"不解析 → 绕过去"**。把 hotspot 区域**物理切出来**变成独立小图，用 `<a><img></a>` 包裹——这是 HTML 最基础结构，**所有 Outlook 客户端都能解析**。

### 数据流

```
原图 (W, H) + N 个 hotspot (x1, y1, x2, y2)
  ↓
提取每个 hotspot 的 X 范围
  ↓
合并去重 X 切割线（不含 Y 切割线）
  ↓
切成 K+1 条竖条（每条横跨完整 Y 范围 [0, H]）
  ↓
每条竖条要么是"普通切片"（<img>）、
要么是"链接切片"（<a><img></a>）
```

### 关键设计

| 维度 | 选择 | 原因 |
|------|------|------|
| 切割方向 | **仅 X 轴**（纵向） | Y 切会破坏"竖条整条可点"（用户期望点哪都能跳） |
| 竖条高度 | **整高 [0, H]** | 链接竖条整条可点，用户点哪都能跳 |
| 重叠 hotspot | **禁止** | V1 架构硬约束，避免歧义 |
| 嵌套 hotspot | **禁止** | 同上 |
| 兜底机制 | **无** | V1 干净，删除 `<map>+<area>` 兜底（之前用户接受"Outlook Desktop 可点"是硬需求） |

### V1 架构硬约束
- ✗ 仅纵向切割（X 轴），不切横向（Y 轴）
- ✗ 不支持重叠 hotspot（X 范围相交 → 拒绝）
- ✗ 不支持嵌套 hotspot（X 范围包含 → 拒绝）
- ✗ 不支持一个竖条挂多个 URL
- ✗ 不支持 `<map>/<area>` 兜底
- ✗ 不支持透明覆盖层

## 核心代码

### `hotspot_slicer.py`

```python
def compute_cut_lines(img_w, hotspots):
    """计算 X 切割线：图片左右边 + 每个 hotspot 的 x1, x2"""
    lines = {0, img_w}
    for h in hotspots:
        lines.add(h.x1)
        lines.add(h.x2)
    return sorted(lines)

def slice_image_with_hotspots(img, hotspots, source_index=0.0):
    """按 hotspot X 范围纵向切割原图"""
    img_w, img_h = img.size

    # 1) 校验
    ok, reason = validate_hotspots_no_overlap(hotspots, img_w)
    if not ok:
        raise ValueError(reason)

    # 2) 计算切割线
    cut_lines = compute_cut_lines(img_w, hotspots)

    # 3) 分配 hotspot 到对应竖条
    assignments = build_stripe_assignments(cut_lines, hotspots)

    # 4) 物理切割（整高竖条）
    stripes = []
    for i in range(len(cut_lines) - 1):
        x1, x2 = cut_lines[i], cut_lines[i + 1]
        stripe_img = img.crop((x1, 0, x2, img_h))  # 整高
        href, text = assignments.get(i, (None, ""))
        stripes.append(CutStripe(
            image=stripe_img, x1=x1, x2=x2,
            y1=0, y2=img_h,  # 整高
            href=href, hotspot_text=text,
            sort_key=source_index + (i + 1) * 0.001
        ))
    return stripes
```

### 重叠检测

```python
def validate_hotspots_no_overlap(hotspots, img_w):
    """
    V1 重叠检测：检查所有 hotspot 的 X 范围是否相交 / 嵌套。
    严格按"禁止重叠"原则，连相邻擦边都允许（边界相接不算重叠）。
    """
    if not hotspots:
        return True, ""
    # 1) 越界
    for i, h in enumerate(hotspots):
        if h.x1 < 0 or h.x2 > img_w or h.x1 >= h.x2:
            return False, "Hotspot 坐标超出图片范围"
    # 2) 重叠 / 嵌套（按 x1 排序后比较相邻）
    sorted_h = sorted(hotspots, key=lambda h: h.x1)
    for i in range(len(sorted_h) - 1):
        a = sorted_h[i]
        b = sorted_h[i + 1]
        if a.x2 > b.x1:  # 边界相接不算（a.x2 == b.x1 允许）
            return False, "Hotspot X 范围重叠"
    return True, ""
```

## 优势 vs 局限

### 优势
- ✓ Outlook Desktop 100% 可点击（仅用 `<a><img></a>`）
- ✓ 图上看不到任何标注（hotspot 边界不渲染）
- ✓ 多 hotspot 支持（任意位置）
- ✓ 切割线模型可扩展到 V2（横向 + 网格）

### 局限
- ✗ V1 物理切割保持整图高度 = 同一原图的多个 hotspot 切出 N+1 条竖条 = 邮件里"多张图"
- ✗ 视觉上没有"按钮感"（无背景色 / 边框 / 圆角）—— 只是"点上去可跳的图"
- ✗ 客户点"非链接竖条"无反应（设计上让链接竖条整高可点 → 点非链接竖条就没反应）
- ✗ "用户期望的按钮" = 图上画个圆角矩形 → V1 无法实现（物理切割的产物是"原图的一部分"）
